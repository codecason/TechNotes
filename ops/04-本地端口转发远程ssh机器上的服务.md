有一个常见的情景，是使用别人给好的ssh服务器，连接上去进行开发；此时如果是纯终端的界面会不太方便；所以我们有个解决方案：使用VSCode的Remote-SSH；但是还有个是在远程服务器起了一个界面服务，应该怎么去看？

所以我们需要解决两个问题：

1.如果我ssh了一个服务器，ifconfig看到的哪个ip是可用的？

2.如果ssh能连上，但是监听端口连不上那是因为什么？有无绕开端口限制的方法？

回答：

1.找到`10.x.x.x`、`172.16.x.x` 到 `172.31.x.x` 或 `192.168.x.x` 开头的IP；一个一个去试；

2.监听端口连不上，是因为防火墙默认不会开发端口的，此时需要做端口转发；

端口转发例子：

```
ssh -p 2222 -L 8080:localhost:7860 user@server_ip
```

p是远程服务的ssh端口；

7860的位置是目标服务器上起的端口；

8080是本地的端口，用来后续连接的，连通之后在本地连接浏览器localhost:8080即可访问服务器上的7860服务；

**如果是密钥验证权限，改成：**

~~~
ssh -p 2222 -L 8080:localhost:7860 user@server_ip -i ~/.ssh/id_rsa_xx
~~~



当然，端口转发是可以在服务端设置为禁用的！遇到这种情况你只能放弃或者请管理员喝茶，因为你不知道这个严苛的设置是不是为了杜绝各种你不知道的高级安全问题。
